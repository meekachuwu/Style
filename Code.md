# Code Format Standards
## Naming Convention
### File System
Folders
	- General:		Capitalized
Files
	- Scripts:		Lowercase
	- Modules:		Capitalized
	- ReadMe:		Capitalized
### Code
Varibles
	- General:		1 word, lowercase, letters only.  If more words needed, nested varibles in objects or object-like structures
## Format
### Comments
Placement
	- General:		Use self explanatory varible names instead of comments, then add short comments to the side of the code summerizing the purpose of the logic in 1 sentance
	- Alignment:	Line up comments with tabs
### Nesting
Limits
	- General:		Minimize nesting by modulating with OOP
Indention
	- Alignment:	Line up code & comments with tabs, especially parts that are identical &/or vertically adjacent
General:
## Syntax
### Restrictions
Standards
	- JavaScript:	Use strict mode
### Keywords
Varibles
	- Declare:		No "var", only "let" or "const"
Conditions
	- General:		Use Switch instead of If statements whenever there is more than 2 cases
Functions
	- General:		Use arrow functions (=>) whenever possible
Loops
	- Arrays:		Use array methods (foreach, every, filter, map) instead of standard loops
Quotes
	- General:		Use template literals (`) instead of quotation marks
